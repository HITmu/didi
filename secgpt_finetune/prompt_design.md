# Prompt 设计文档

## 设计理念

本项目的 Prompt 体系围绕三个核心原则设计：

### 1. 角色锚定（Role Anchoring）

每个 prompt 都以明确的系统角色开头，为模型建立稳定的回答框架：

```
你是一个网络安全专家，擅长分析API安全漏洞和生成安全报告。
```

**设计理由：**
- 安全领域知识高度专业化，需要模型在特定领域内回答
- 角色锚定能减少模型"出戏"（回答偏离安全范畴）
- 一致的system prompt确保Base和LoRA模型在相同角色设定下对比

### 2. 任务纯化（Task Purification）

测试 prompt 被设计为纯问题形式，避免引入额外上下文干扰：

```
# 好的设计（纯净任务）
解释SQL注入攻击的原理

# 应避免的设计（引入干扰）
基于以下日志分析SQL注入攻击：[日志内容]
```

**设计理由：**
- 目标是评估模型自身的安全知识，而非上下文理解能力
- 排除RAG上下文的干扰变量，聚焦于模型微调本身的效果
- 后续RAG场景可在此基础上叠加上下文

### 3. 覆盖广度（Coverage Breadth）

测试问题覆盖 30 个安全子领域，确保评估的全面性：

| 领域分类 | 覆盖数量 | 说明 |
|---------|---------|------|
| 经典Web注入 | 62 | SQL/XSS/NoSQL/命令注入 |
| API认证授权 | 60 | OAuth/JWT/IDOR/RBAC |
| 架构安全 | 55 | API网关/CORS/WebSocket/GraphQL |
| 防御技术 | 52 | 速率限制/加密/安全测试 |
| 合规标准 | 23 | OWASP/PCI DSS/GDPR |
| 云原生安全 | 36 | 容器/K8s/Serverless/云API |

## System Prompt 设计

```
你是一个网络安全专家，擅长分析API安全漏洞和生成安全报告。
```

**设计要素：**

| 要素 | 内容 | 作用 |
|------|------|------|
| 角色 | 网络安全专家 | 锚定专业领域 |
| 专长 | API安全漏洞分析 | 明确定义能力范围 |
| 产出 | 安全报告 | 定义输出格式风格 |

**为什么选择中文 Prompt？**
- 训练数据（QA对）全部为中文
- 模型测试在中文场景下进行
- 避免中英混杂导致的输出不稳定

## 评估 Prompt 设计（LLM API 评委）

### 评分维度

| 维度 | 评分范围 | 评估标准 |
|------|---------|---------|
| 准确性（Accuracy） | 1-10 | 技术内容是否正确，是否存在幻觉或错误 |
| 完整性（Completeness） | 1-10 | 回答是否覆盖问题的关键要点 |
| 结构化（Structure） | 1-10 | 回答的组织方式是否清晰、层次分明 |
| 专业性（Professionalism） | 1-10 | 术语使用是否准确，分析深度是否足够 |

### 评分设计理由

1. **准确性优先**：安全领域回答错误比回答不完整危害更大
2. **完整性次之**：确保回答不是"正确的废话"
3. **结构化是 LoRA 的主要改进点**：从 Base 模型的平铺直叙到结构化的层次输出
4. **专业性区分领域深度**：区分"网络安全常识"和"专业安全分析"

### LLM 评委配置

```
model: gpt-4o
temperature: 0.1   # 低温确保评分一致性
max_tokens: 6000    # 足够输出20条详细评分
```

**低温度（0.1）的理由：**
- 评分任务需要一致性而非创造性
- 减少评分波动，提高可比性
- Base vs LoRA 对比需要稳定的评判标准

## 对比方法论

### 控制变量

| 变量 | 控制方式 |
|------|---------|
| 输入 Prompt | Base 和 LoRA 使用完全相同的 system + user prompt |
| Generation 参数 | temperature=0.3, top_p=0.9, max_new_tokens=128, repetition_penalty=1.1 |
| 评估标准 | LLM 对两个模型的输出使用相同的评分维度 |

### 统计方法

- **定量**：平均回答长度、空回答率、LLM 四维评分
- **定性**：LLM 对每个样本给出 winner 判定和理由
- **汇总**：Base wins / LoRA wins / Tie 分布
